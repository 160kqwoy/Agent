# 文档质量检查规范 (Quality Checklist)

## 4.1 三级检查体系

### 一级：完整性检查 (Completeness)
- [ ] 标题是否包含系统名称"Memoria"？
- [ ] 开场是否吸引注意力并说明项目动机？
- [ ] 每个提纲节点都有对应内容？（无悬空标题）
- [ ] 是否包含个人学习收获？
- [ ] 关键数据是否标注来源？

### 二级：一致性检查 (Consistency)
- [ ] 术语全文统一（如"用户"与"客户"不混用）
- [ ] 数据前后一致（如前文说"8大痛点"，后文枚举是否为8个）
- [ ] 时态统一（回忆过去用过去时，现状用现在时）
- [ ] 人称统一（保持"我们"指代团队）
- [ ] 标点规范（中文正文用全角标点）

### 三级：有效性检查 (Effectiveness)
- [ ] 删除测试：删除任意一段是否损害整体理解？如否，则冗余
- [ ] 反向阅读测试：从结论倒推，论据是否足够支撑？
- [ ] 时间测试：语速270字/分钟，总时长是否≤3分钟？
- [ ] 字数测试：总字数是否≤900字？
- [ ] 学生身份测试：是否体现学生汇报的特点（学习收获、成长）？

## 4.2 学生汇报特化检查项
- [ ] 口语化程度：是否适合口头表达（避免书面化长句）？
- [ ] 停顿点设置：是否有合理的呼吸停顿？
- [ ] 重点突出：关键数据和结论是否明确？
- [ ] 时间分配：各部分时长是否合理？
- [ ] 学习收获：是否体现个人成长和技术提升？

## 4.3 评分与迭代机制
- **基线分**：满分100，一级检查每项15分，二级每项10分，三级每项5分
- **通过标准**：≥85分可输出；70-85分需标注待改进项；<70分重写

## 4.4 常见失败模式 (Anti-patterns)
| 模式名称 | 症状 | 修复策略 |
|---------|------|---------|
| 头重脚轻 | 背景写太多，结论太简短 | 强制结论区字数≥开场×1.5 |
| 数据堆砌 | 罗列数据无分析 | 每个数据点必须附带解读句 |
| 术语轰炸 | 堆砌专业术语无解释 | 每个术语必须跟"通俗来说..."的解释 |
| 超时风险 | 内容过长超过3分钟 | 严格按字数分配表裁剪内容 |
| 逻辑跳跃 | 缺少过渡直接跳转 | 强制使用"接下来"、"具体而言"等连接词 |
| 缺少收获 | 只讲技术不讲学习 | 必须包含个人学习收获章节 |
